> 版本：v0.2
> 更新：2026-06-16
> 状态：需求梳理中，部分概念待确认

---

## 1. 项目概述

### 1.1 项目背景
本系统是一个**招生录取辅助工具**，用于在一份完整的考生成绩总表中，按照自定义的招生计划（目标学校、专业、名额）对考生进行分组、排序、筛选，并**按目标学校**导出多张分组结果表，辅助招生人员进行录取决策。

### 1.2 目标用户
招生工作人员 / 录取管理人员。

### 1.3 核心价值
- 把"一份大表 → 多张分学校小表"的手工流程自动化。
- 用统一规则（分数下限 + 多级排序 + 名额分配）保证录取公平、可复核。
- 纯前端、零后端依赖，数据本地化，便于在离线/内网环境使用。

---

## 2. 名词与概念约定

| 概念 | 说明 |
| --- | --- |
| **分组（= 目标学校）** | 本系统的顶层组织维度。一个分组对应一所目标学校的完整招生计划。导出时每个分组产出一张表。分组可自定义。 |
| **招收目标人数** | 分组（学校）层面的总招收名额上限。 |
| **专业** | 隶属于某个分组（学校）的招生专业，每个专业单独设置招收人数（名额）。 |
| **考生** | 成绩数据表中的一行记录。 |

### 2.1 数据层级

```
分组（= 一所目标学校的招生计划）
├── 招收目标人数：N
├── 专业 A（招收 15 人）
├── 专业 B（招收 10 人）
└── 专业 C（招收 10 人）

分组（= 另一所学校）
├── 招收目标人数：M
└── ...
```

## 3. 功能需求

### 3.1 数据导入（文件上传页）
- 支持导入一份完整的考生成绩数据表（Excel / CSV）。
- 导入时校验表头与字段类型，缺失/异常字段给出提示。
- 导入成功后，原始数据写入 **IndexedDB**，可在前端持久化保留。
- 支持重复导入（覆盖或合并，策略待定）。

### 3.2 数据筛选规则
- **总分下限**：总分 < 180 的考生直接过滤，不参与任何分组。
- **排序规则**（降序，多级回退）：
  1. 总分
  2. 语文
  3. 数学
  4. 英语
- 排序在**每个分组内**独立进行。

### 3.3 分组与名额配置（配置页）
配置页是一个**管理后台式**的页面，负责维护以下内容：

1. **分数类型配置**
   - 参与排序的科目及其优先级（默认：总分 → 语文 → 数学 → 英语）。
   - 总分下限阈值（默认 180，可调整）。
2. **分组（目标学校）配置**
   - 新增 / 编辑 / 删除分组（即新增一所目标学校）。
   - 为每个分组设置招收目标人数。
3. **专业与名额配置**
   - 在分组下新增 / 编辑 / 删除专业，设置每个专业的招收人数。

> 📌 配置数据同样存入 IndexedDB，与成绩数据分离存储。

### 3.4 录取分配（草案，待确认）
- 在每个分组内，对通过分数下限的考生按多级排序得出名次。
- 按分组内各专业的名额，从高名次到低名次依次分配。
- **待确认问题**：考生与"专业"的对应关系如何确定？
  - 方案 A：考生不指定专业志愿，仅按学校名额"截取"前 N 名（专业名额仅作展示/统计）。
  - 方案 B：考生在数据表中有专业志愿字段，按志愿 + 名额匹配。
  - 当前数据表结构（见第 4 节）无志愿字段，倾向于**方案 A**。

### 3.5 数据展示与筛选页
- 以表格形式展示当前数据（导入的原始 / 筛选后 / 分组后）。
- 提供筛选条件：分组、专业、分数区间、是否达标等。
- 提供排序展示（默认按多级排序规则）。
- 展示每个分组（学校）/专业的名额与已分配人数对比。

### 3.6 数据导出
- 按**目标学校（分组）**导出，每个分组产出一张数据表。
- 导出表格的**列结构与导入表一致**（排名、姓名、性别、年龄、总分、语文、数学、英语、身份证号、考号）。
- 支持一次性导出全部分组（打包 / 多 sheet / 多文件，形式待定）。
- 导出格式：Excel（.xlsx）为主，可考虑同时支持 CSV。

## 6. 技术栈与非功能需求

| 项 | 选型 |
| --- | --- |
| 项目类型 | 纯前端（SPA） |
| 框架 | Vue 3（Composition API） |
| 语言 | TypeScript |
| 包管理 | pnpm |
| 样式 | SCSS + Tailwind CSS |
| 本地存储 | IndexedDB（建议封装库：如 `idb` 或 `Dexie.js`） |
| 路由 | Vue Router |
| 状态管理 | Pinia（如需） |
| 表格导入/导出 | Excel：`SheetJS (xlsx)` / `exceljs` |

### 6.1 数据安全
- 所有数据仅存于浏览器本地，不上传任何服务器。
- 提供"清空本地数据"入口。

### 6.2 性能
- 考生数据量级假设为千~万级，前端排序与筛选需保证流畅（IndexedDB 索引 + 分页/虚拟滚动）。

## 7. 待确认 / 开放问题

1. **录取分配逻辑**：考生是否带"专业志愿"字段？还是纯按学校名额截取前 N 名（方案 A）？
2. **导出形态**：多张表以什么形式交付——多个 .xlsx 文件打包 / 一个 .xlsx 多 sheet / 多个 CSV？
3. **重复导入策略**：再次导入时是覆盖旧数据，还是合并？
4. **总分下限 180** 是否需要可在配置页调整？（当前假设：可配置，默认 180）
5. **排序科目优先级** 是否需要可在配置页调整？（当前假设：可配置）
6. **数据表字段** 是否还有其他字段（如生源学校、联系方式、志愿等）？拿到真实样例后需校准。
