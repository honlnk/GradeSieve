# GradeSieve 成绩筛分系统

招生录取辅助工具：导入一份含三张工作表的考生数据文件（同报志愿名单 / 普高拟录取名单 / 招生计划），按招生计划对各中职学校从高分到低分录取，并按学校导出录取名单。

纯前端项目，所有数据存储在浏览器本地（IndexedDB），不上传任何服务器。

## 技术栈

- Vue 3 + TypeScript（Composition API）
- Vite 6
- Pinia（状态管理）+ Vue Router
- SCSS + Tailwind CSS
- Dexie.js（IndexedDB 封装）
- SheetJS（Excel 导入/导出，支持 .xls / .xlsx）
- 包管理：pnpm

## 快速开始

```bash
pnpm install      # 安装依赖
pnpm dev          # 启动开发服务器（默认 http://localhost:8989）
pnpm build        # 类型检查 + 生产构建
pnpm preview      # 预览构建产物
```

## 功能

- **数据导入页** (`/upload`)：上传含三表的 .xls/.xlsx，解析后写入本地数据库。
- **录取结果页** (`/data`)：查看录取流程摘要、各校录取进度、按学校/状态/关键字筛选考生名单，一键导出各校录取名单。
- **招生计划页** (`/plan`)：查看表3 招生计划，调整录取规则（语数外门槛）。

## 录取业务规则

输入文件需包含三张工作表：

| 表 | 内容 | 列结构 |
| --- | --- | --- |
| 表1 | 普职班同报志愿名单 | 序号、区县、就读学校、报名号、姓名、性别、身份证号、民族、学籍号、普高1、普高2、**职教高考班**、3+4志愿学校、语文、数学、英语、物理、化学 |
| 表2 | 普高拟录取名单 | 同表1 |
| 表3 | 各校招生计划 | 学校名称（`_代码_校名`）、计划数 |

录取算法：

1. **剔除已被普高录取的考生**：表1 中身份证号出现在表2 的考生（普高已录取）不参与中职录取。
2. **语数外门槛**：剩余考生中，语文+数学+英语 总分低于门槛（默认 **205** 分）的剔除。
3. **按志愿分组**：考生按其"职教高考班"志愿所属学校分组（唯一志愿，不调剂）。
4. **校内排序**：每所学校内按 语数外总分 → 语文 → 数学 → 英语 降序排序。
5. **按计划截取**：各校按表3 计划数从高分到低分录取，超出者列为候补。

> 导出时每所学校产出一个 sheet，仅含录取考生，列含校内名次、姓名、身份证号、各科成绩、语数外总分等。
